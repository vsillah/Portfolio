# Fable 5 production line-edit brief

Use this brief with Claude Code agent `Fable 5` for the production-shaping pass of `Agentified`.

## Objective

Line edit selected second-draft chapters for rhythm, clarity, and production readiness.

This is not a rewrite. Preserve the chapter title, chapter argument, Sam's story beats, framework, safety boundaries, and approximate length. Improve read-aloud flow, transitions, sentence rhythm, and chapter ending strength.

## Voice

Write in Vambah's register:

- grounded, reflective, practical, morally clear, and systems-minded
- scene first, system diagnosis second, practical operating tool third
- clear product-management thinking without consulting filler
- human stakes without melodrama
- plain strong sentences
- a bridge between survival-shaped communities and technology-shaped futures

## Line-edit targets

1. Read-aloud rhythm.
   - Cut stiff phrasing.
   - Vary sentence length.
   - Remove repetitive transitions.
   - Keep paragraphs breathable.

2. Structural clarity.
   - Make the chapter's opening tension easy to feel.
   - Make the upgraded `Accelerated` principle clear.
   - Keep the framework usable.
   - Make the ending land as a principle, question, or next operating move.

3. Safety and source discipline.
   - Agents prepare, recommend, route, evaluate, summarize, and package evidence.
   - Humans approve publishing, spending, memory promotion, client claims, irreversible action, external sends, and consequential decisions.
   - Shaka is the Chief of Staff/router identity only.
   - Treat named client/customer scenes as fictional or composite unless the manuscript explicitly says otherwise.
   - Do not invent metrics, dates, exact counts, timelines, customers, outcomes, or precise timestamps.

4. Production polish.
   - Remove visible "draft" language.
   - Avoid repetitive bolded mini-headers unless the chapter already depends on them.
   - Preserve Markdown heading structure.
   - Use ASCII punctuation only.

## Output rules

- Return only the edited chapter in Markdown.
- Start with the exact same first-line heading.
- Do not include notes, summaries, alternatives, caveats, or commentary.
- Do not add a new chapter title.
