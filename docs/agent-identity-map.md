## Operating Rules

- Keep technical keys stable unless a planned migration updates routes, tests, database records, Slack commands, and docs together.
- Use the historical identity as the visible name and the functional responsibility as the subtitle or suffix.
- Treat these as operating names, not claims of endorsement by any person, people, lineage, or institution.
- If a future identity needs deeper public-facing provenance, verify the historical fit before promoting it outside internal admin surfaces.
